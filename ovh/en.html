In this tutorial, we'll guide you through the process of implementing a "Round Robin" configuration for an AWS Cloud Load Balancer using the AWS API. Round Robin is a simple load balancing algorithm that distributes requests evenly between several backend servers. You will learn how to create an Application Load Balancer (ALB) with a Round Robin configuration.

Prerequisites

You'll need the following

  1. An active AWS account.
  2. AWS CLI (Command Line Interface) installed and configured with your AWS credentials.
  3. Basic knowledge of AWS and Load Balancers.

Creating a JSON Configuration File #back to top

Create a JSON file named load-balancer-config.json with the following contents:

                                    {
                                        "Name": "mon-load-balancer",
                                        "Subnets": ["subnet-12345678", "subnet-98765432"],
                                        "SecurityGroups": ["sg-0123456789abcdef0"],
                                        "Scheme": "internet-facing",
                                        "Listeners": [
                                            {
                                                "Protocol": "HTTP",
                                                "Port": 80,
                                                "DefaultActions": [
                                                    {
                                                        "Type": "fixed-response",
                                                        "FixedResponseConfig": {
                                                            "ContentType": "text/plain",
                                                            "StatusCode": "200",
                                                            "Content": "OK"
                                                        }
                                                    }
                                                ]
                                            }
                                        ],
                                        "IpAddressType": "ipv4"
                                    }

Make sure you replace the Subnets and SecurityGroups key values with the appropriate subnets and security groups for your AWS account.

Load Balancer creation #back to top

Open your terminal and run the following AWS CLI command to create the load balancer using the JSON configuration file you created:

                            aws elbv2 create-load-balancer --cli-input-json file://load-balancer-config.json

This will create an Application Load Balancer (ALB) with the configuration parameters specified in the JSON file.

Round Robin load balancer configuration #back to top

Now that the load balancer has been created, we're going to configure Round Robin load balancing for the backend servers. To do this, we'll create a target group and associate EC2 instances (or other resources) with this group.

Create a JSON file named target-group-config.json with the following contents :

                            {
                                "Name": "mon-groupe-de-cibles",
                                "Protocol": "HTTP",
                                "Port": 80,
                                "TargetType": "instance",
                                "Targets": [
                                    {
                                        "Id": "i-0123456789abcdef0"  // Remplacez par l'ID de votre instance EC2
                                    },
                                    {
                                        "Id": "i-9876543210fedcba9"  // Ajoutez d'autres ID d'instances si nécessaire
                                    }
                                ],
                                "Matcher": {
                                    "HttpCode": "200"
                                }
                            }

Make sure you replace the Id key values with the IDs of your EC2 instances.

Run the following AWS CLI command to create the target group:

                        
                            aws elbv2 create-target-group --cli-input-json file://target-group-config.json

Associate the target group with the load balancer using the following command:

                        
                            aws elbv2 create-rule --listener-arn "arn:aws:elasticloadbalancing:us-east-1:123456789012:listener/app/mon-load-balancer/1234567890abcdef0/4567890abcdef012" --priority 1 --action-type "forward" --target-group-arn "arn:aws:elasticloadbalancing:us-east-1:123456789012:targetgroup/mon-groupe-de-cibles/abcdef01234567890"

Be sure to replace the ARN (Amazon Resource Name) values with the ARNs for your load balancer and target group.

Conclusion #back to top

You have now configured a Cloud AWS Load Balancer with Round Robin load balancing. Requests will be equally balanced between the instances associated with the target group. This configuration optimizes the availability and performance of your applications. You can extend this configuration by adding more instances to the target group as your traffic increases.
